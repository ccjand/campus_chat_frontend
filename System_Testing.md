# 第5章 系统测试

## 5.1 测试环境与测试方法

为了保证本校园即时通讯与管理系统的稳定性和功能的正确性，在系统开发完成后进行了全面的系统测试。
系统测试环境包括：

- **硬件环境**：Apple Mac，内存 16GB，硬盘 512GB。
- **软件环境**：操作系统 macOS，后端环境 JDK 1.8，数据库 MySQL 8.0，缓存 Redis，消息队列 RocketMQ，前端运行环境 Node.js 与微信开发者工具。
- **测试方法**：采用黑盒测试方法，主要通过等价类划分法和边界值分析法，对系统的各个核心功能模块进行功能验证，同时验证前后端数据交互与业务流转是否符合预期设计。

## 5.3 测试总结

经过对本校园即时通讯与管理系统的核心模块（用户管理、即时通讯、课堂签到、请假审批、通知与考试）进行系统化测试，各功能模块的业务逻辑流转均符合预期。
其中：

1. **即时通讯模块**中，WebSocket长连接稳定，RocketMQ离线消息补推机制在用户重连时触发准确，无消息丢失现象。
2. **课堂签到模块**的Haversine球面距离计算准确，能有效识别定位签到的作弊行为。
3. **请假及审批模块**状态机流转清晰，数据一致性良好。
4. **通知与考试模块**前后端交互顺畅，数据展示与已读状态更新准确无误。

综上所述，系统各项测试指标均达到设计要求，系统运行稳定，具备了上线部署的条件。
